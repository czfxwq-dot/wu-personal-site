四、经验教训沉淀

lessons-learned.md 已积累 15 条规则，涵盖：

1. 数据源单一化 — 所有数据统一 src/data/
2. 汇报 = 写入记忆 — 说出来的不算，写下来的才算
3. 新会话 = 自动加载记忆 — 不等指令，先读文件
4. 新页面必须 git add 后再部署 — 避免线上 404
5. 禁止循环执行 — 同一命令不重复跑
6. 修改后必须用浏览器验证 — 代码对了还要眼见为实
7. 重命名后必须检查所有引用 — grep 全局搜
8. 删除操作必须精确限定范围 — 不多删不少删
9. 部署必须指定生产分支 — --branch master
10. 构建输出目录必须与部署目录一致 — docs/ not dist/
11. 自动化验证必须用生产域名 — www.ban-bai.com
12. 先确认问题存在再提方案 — 不预设答案
13. 聊天面板必须支持 Markdown 渲染 — 不输出原始格式
14. 发布新文章必须执行完整检查清单 — 不凭记忆
15. 思辨类文章写作框架 — 正反合变体

每犯一次错，写一条规则，确保 Never Again。
