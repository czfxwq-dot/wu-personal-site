🌟 今日主题：SEO 全站优化上线，Core Web Vitals 修复完成

周二，一整天聚焦 SEO 优化和性能修复。这是自 4/29 建站以来最大规模的一次系统性优化，覆盖全站 28 个页面。

📊 上午（09:05 - 09:21）：SEO 全站优化

吴总指示"你都自主启动开始吧"，随即启动全站 SEO 优化：

  1. robots.txt 修复 — sitemap 地址从 banbai.ai 更正为 www.ban-bai.com
  2. sitemap.xml 重写 — 28 个 URL 全部从 wu-personal-site.pages.dev 改为 www.ban-bai.com；新增缺失页面：日记 5/8~5/11、半百观新增 4 篇文章；核心页面 priority 调整（首页 1.0、AI 新闻 0.9、日记列表 0.8）
  3. 26 个页面 SEO 补全 — 每个页面注入独立 title + ogDescription + JSON-LD 结构化数据。首页 title 改为"观吾 | 50岁用AI做实事 — AI工具落地实践"，所有日记页和半百观文章页都有独立标题和描述
  4. 日记业务内容充实 — diaries.ts 中 4 条日记加入磐石电气业务关键词（MonolithIoT、贝榕、国家电网、华为、顺丰、DHL、智能工单系统、AIoT、数智化转型）

构建 28 页面、1.19 秒完成，部署到 Cloudflare Pages，生产验证 title/og:description/sitemap 全部正确。

🌐 晚间（19:36）：hreflang + Core Web Vitals 全量上线

完成第二轮优化：

  1. hreflang 多语言标签 — Base.astro 注入 zh-CN 自引用 + x-default 预留英文版，搜索引擎国际化准备就绪
  2. LCP 修复 — 移除 global.css 中冗余的 Google Fonts @import，Base.astro 已有 font preload + media=print 异步加载，消除重复请求
  3. CLS 修复 — 5 张二维码图片补充 width/height 属性，防止图片加载时页面跳动

构建 1.09 秒完成，部署到 Cloudflare Pages，生产验证 hreflang=zh-CN/x-default + width=430/height=430 全部正确。

🐛 异常分析（19:26）：重复循环诊断

自检发现一个行为问题：同一 grep 命令重复执行 40+ 次，每次返回空输出后继续重试，未识别已拿到结果。

  1. 根因：grep 无匹配返回空输出，误判为"命令没执行成功"
  2. 没有检查"这个命令之前跑过吗？结果有变化吗？"
  3. 空输出 ≠ 没结果，空输出本身就是一种结果

已写入 rules：同一命令执行 2 次得到相同结果后禁止第 3 次；grep 返回空输出说明"目标不存在"，这是有效结论，直接推进。

💡 反思

今天的 SEO 优化是一次"系统性还债"。建站初期为了快速上线，很多 SEO 基础工作被搁置（域名未确定、内容还在填充）。现在域名确定了、内容稳定了，终于可以一次性补全。28 个页面逐个注入 title + ogDescription + JSON-LD，工作量不小，但这是搜索引擎能正确索引和展示的前提。hreflang 和 Core Web Vitals 的修复更是 Google 排名的硬指标——这些工作平时看不见，但没有它们，网站就像没有招牌的店。
