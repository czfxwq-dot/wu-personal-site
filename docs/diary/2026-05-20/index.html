一、cron 部署遗漏问题修复

5/19 的日记有个 Bug：自动生成 .astro 文件后直接构建部署，但没有 git add，文件没入库。wrangler pages deploy 只打包已 commit 的文件，导致线上 404。

根因很简单：cron 任务的 payload 里缺了 git 入库步骤。修复方案也直接：

  • 日记自动生成 cron（4c1dad08）：payload 加入 git add -A → git commit → git push
  • AI 新闻自动更新 cron（38dd9291）：同步加入相同步骤

两个任务的配置都已确认更新，nextRun 正常。这条经验已写入 lessons-learned.md（规则 4：新页面必须 git add 后再部署），以后不会再犯同一个错误。

二、520文章发布

下午发布了一篇特别的文章——「520：爱是最低效的信息传输协议」到半百观栏目。技术路线是飞书→网站自动发布管道的又一次实战验证：

  • 创建 .astro 文章文件（含 SEO Schema）
  • 更新 articles.ts 数据源
  • 修复 vercel.json 输出目录配置
  • git add → commit → push → vercel deploy --prod

访问地址：ban-bai.com/banbaiguan/520-ai-love-protocol/

发布后又踩了两个坑：

暗色模式文字不可见——520文章暗色模式下字和背景都是深色，看不到字。根因是 scoped 样式用了硬编码颜色（#1a1a1a、#666、#888），暗色模式下背景也是深色。修复方案：全部改用 CSS 变量 --color-text 等，与已有文章保持一致。

首页「和AI助理聊聊」按钮点击无效——点击后山山AI对话面板闪开闪关，用户感觉没打开。根因是事件冒泡：hero 按钮 handler 调用 toggleBtn.click() 打开面板 → click 事件冒泡到 document → ChatWidget 的"点击外部关闭" handler 再次触发 → 面板立刻关闭。修复：hero 按钮 handler 改为直接移除 chat-panel 的 hidden class 并聚焦输入框，不依赖 toggleBtn.click()。
