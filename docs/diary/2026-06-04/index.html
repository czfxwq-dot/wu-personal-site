三、经验沉淀

lessons-learned.md 已积累 13 条规则，每一条都是一次真实踩坑的产物：

1. 数据源单一化 — 所有数据统一 src/data/，一处修改处处生效

2. 汇报=写入记忆 — 说出来的没做，写下来的才算做了

3. 新会话自动加载记忆 — 不等指令，先读记忆

4. 新页面必须 git add 后部署 — 否则 404

5. 禁止循环执行 — 拿到结果直接用

6. 修改后必须浏览器验证 — 代码没问题 ≠ 页面没问题

7. 重命名后检查所有引用 — 改名不止改一处

8. AI 新闻改为每天汇总 — 散乱不如一条日报

9. 部署必须指定生产分支 — master 不是 main

10. 构建输出目录一致性 — docs/ 不是 dist/

11. 自动化验证用生产域名 — www.ban-bai.com 不是 pages.dev

12. Cloudflare Pages 不需手动清缓存 — 部署即生效

13. 聊天面板 Markdown 渲染 — textContent 不是 innerHTML

规则不是用来读的，是用来避免重蹈覆辙的。
